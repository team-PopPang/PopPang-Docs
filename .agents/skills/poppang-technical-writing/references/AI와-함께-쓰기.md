# AI와 함께 쓰기

이 문서는 `technical-writing.dev/tutorial/review-prompt.html`을 바탕으로, 새 스킬이 AI 보조 작성과 프롬프트 작업을 어떻게 다뤄야 하는지 정리한 문서다.

## source page

- `https://technical-writing.dev/tutorial/review-prompt.html`

## 이 페이지가 다루는 것

- GPT 지침을 붙여서 전용 봇을 만드는 흐름
- Step 1, Step 2, Step 3에 맞는 프롬프트 사용 위치
- 문서 유형 추천, 구조 피드백, 문장 다듬기 요청을 AI에 맡길 때의 입력 형식

## 새 스킬에서 지켜야 할 규칙

- source page에 프롬프트 전문이 있으면 그대로 사용한다.
- 프롬프트를 토스식 문체로 바꿔 쓰지 않는다.
- 프롬프트 바깥 설명과 사용법만 토스식으로 정리한다.
- 프롬프트를 요약본으로 바꾸지 않는다.

## 프롬프트 계열

### Step 1. 문서 유형 정하기

- 역할: 문서 목표, 독자 수준, 프로젝트 상황, 추가 고려 사항을 바탕으로 가장 적합한 문서 유형을 고르게 한다.
- 출력 기대:
  - 가장 적합한 유형
  - 작성 시 주의할 점
  - 필요 시 복수 유형 제안

### Step 2. 정보 구조 만들기

- 역할: 문서 초안이나 목차를 보고 구조적 문제와 개선안을 제안하게 한다.
- 핵심 원칙:
  - 한 페이지 하나의 목표
  - 개요 제공
  - 예측 가능한 구조
  - 가치 우선
  - 효과적인 제목

### Step 3. 문장 다듬기

- 역할: 입력 문장을 더 짧고 명확하게 다듬게 한다.
- 핵심 원칙:
  - 필요한 정보만 남기기
  - 메타 담화 줄이기
  - 구체적으로 쓰기
  - 일관되게 쓰기
  - 문장의 주체를 분명하게 하기

## 언제 이 문서를 읽나

- 사용자가 GPT 지침을 원할 때
- 사용자가 프롬프트 원문을 그대로 달라고 할 때
- 기술 문서용 봇 프롬프트를 만들거나 옮겨 적어야 할 때
- AI에게 구조 피드백이나 문장 피드백을 맡기는 형식을 만들 때

## 작업 절차

1. 사용자가 원하는 프롬프트 종류를 고른다.
2. source page에서 해당 프롬프트 블록을 확인한다.
3. 프롬프트 본문은 수정하지 않는다.
4. 필요한 경우 앞뒤 사용 안내만 짧게 덧붙인다.
