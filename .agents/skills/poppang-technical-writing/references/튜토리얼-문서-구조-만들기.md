# 튜토리얼 및 문서 구조 참고

이 문서는 `technical-writing.dev`의 튜토리얼 계열 중 `기본 문서 작성하기`와 `문서 구조 만들기` 내용을 작업용으로 요약한 문서다.

## 기본 문서 작성하기

source:

- `tutorial/basic-documents.html`

핵심은 전후 비교다.
좋은 튜토리얼과 좋은 문제 해결 문서는 결과물보다 독자의 진행 경험을 먼저 설계한다.

### 학습 문서에서 봐야 할 것

- 목표가 첫 화면에서 드러나는가
- 준비물과 사전 조건이 빠지지 않았는가
- 단계가 자연스럽게 이어지는가
- 예제가 실제로 실행되는가
- 갑작스러운 개념 점프가 없는가

### 문제 해결 문서에서 봐야 할 것

- 어떤 문제를 해결하는지 즉시 보이는가
- 원인과 해결 절차가 구조적으로 정리되어 있는가
- 코드가 먼저 튀어나와 인지 부하를 만들지 않는가
- 해결 후 확인 방법이 보이는가

## 문서 구조 만들기

source:

- `tutorial/structure.html`

여러 페이지로 된 문서군을 다룰 때는 아래 세 가지를 먼저 본다.

1. 문서 유형이 맞는가
2. 문서 흐름이 논리적인가
3. 관련 문서를 연결하는 크로스링크가 충분한가

문서 구조에는 정답이 없다.
하지만 독자가 무엇을 먼저 읽어야 하는지는 명확해야 한다.

## 문서군 설계 체크리스트

- 상위 문서가 전체 개요를 맡고 있는가
- 상세 문서가 질문 단위로 분리되어 있는가
- 페이지마다 역할이 하나씩만 있는가
- 다음에 읽을 문서를 자연스럽게 연결했는가
- 설치, 개념, 작업 절차, 오류 해결, 참조를 한 페이지에 몰아넣지 않았는가

## 이 문서를 읽어야 하는 상황

- 사용자가 여러 문서로 나뉜 구조를 원할 때
- 기존 문서가 한 페이지에 너무 많은 내용을 담고 있을 때
- 튜토리얼 흐름과 문서군 구조를 동시에 설계해야 할 때
- 독자를 다음 문서로 어떻게 안내할지 정해야 할 때
