# Technical Writing

## 목적

이 스킬은 `technical-writing.dev`의 `시작하기`, `튜토리얼`, `AI와 함께 쓰기`, `Step 1~3` 흐름을 바탕으로 기술 문서 작업을 수행한다.
다음 4가지를 하나의 흐름으로 처리한다.

- 어떤 문서 유형을 써야 하는지 결정
- 기존 초안의 정보 구조를 개선
- 기존 문장을 더 명확하고 자연스럽게 다듬기
- 원문이 없으면 주제만 받아 새 문서 초안 생성

## 언제 이 스킬을 써야 하나

- 사용자가 기존 문서를 더 좋게 다듬고 싶을 때
- 어떤 문서 유형으로 써야 할지 먼저 판단하고 싶을 때
- 문서 구조가 뒤섞여 있어 재구성이 필요할 때
- 문장은 있는데 기술 문서답게 정리되지 않았을 때
- 주제만 있고 빠르게 학습용 문서 초안이 필요할 때
  - 예: `flutter 사용법 문서 만들어줘`

## 참조 문서 읽기 순서

이 스킬이 트리거되면 아래 기준 문서를 사용한다.

1. 항상 `references/문서유형정하기.md`를 기준으로 문서 유형을 정한다.
2. 구조 개선이나 새 초안 생성이 필요하면 `references/정보-구조-만들기.md`를 따른다.
3. 문장 수정이나 최종 정리가 필요하면 `references/문장-다듬기.md`를 따른다.
4. 예시 톤이나 초안 패턴이 필요하면 `references/examples/` 아래 예시 문서를 참고한다.

## 입력 해석 규칙

### 다듬기 모드

아래 중 하나라도 있으면 다듬기 모드로 판단한다.

- 기존 문서 본문이 있다
- 문단/문장 초안이 있다
- 파일 경로와 함께 `다듬어줘`, `리뷰해줘`, `구조 바꿔줘` 같은 요청이 있다

### 생성 모드

아래 조건이면 생성 모드로 판단한다.

- 주제만 있고 본문 초안이 없다
- 빠른 학습용 설명이나 입문 문서가 필요하다
- `문서 만들어줘`, `가이드 작성해줘`, `초안 써줘` 같은 요청이다

## 시작하기와 튜토리얼 관점에서의 기본 판단

사용자가 처음 배우는 내용을 빠르게 따라 해야 하는 상황이면 `시작하기 가이드`나 `튜토리얼`로 구체화한다.

- 설치, 환경 설정, 첫 실행, 가장 빠른 입문이면 `시작하기 가이드`
- 직접 따라 만들면서 익히는 학습이면 `튜토리얼`
- 작업 목표가 오류 해결이면 `문제 해결 가이드`
- API, 옵션, 속성 탐색이 목적이면 `참조 문서`
- 배경, 원리, 비교, 철학 설명이 목적이면 `깊은 이해를 위한 설명 문서`

## 문서 유형 선택지

사용자가 문서 유형을 직접 선택하고 싶어 하거나, 아직 타입이 분명하지 않으면 아래 선택지를 먼저 보여준다.

1. 시작하기 가이드
2. 튜토리얼
3. 문제 해결 가이드
4. 참조 문서
5. 깊은 이해를 위한 설명 문서
6. 추천해줘

`추천해줘`는 항상 마지막에 둔다.

사용자가 타입을 고르지 않았더라도 맥락이 충분하면:

1. 추천안을 먼저 제시한다.
2. 이유를 짧게 설명한다.
3. 사용자가 `알아서`, `추천해줘`, `바로 진행해줘`라고 했으면 추천안으로 계속 진행한다.

## 실행 순서

### 1. 문서 목표와 독자 파악

가능하면 아래를 먼저 정리한다.

- 문서 목표
- 독자 수준
- 프로젝트 상황
- 추가 제약(OS, 버전, 시간, 이미지 활용 여부)

정보가 없으면 최소한의 합리적 가정을 세운 뒤 그 가정을 밝힌다.

### 2. 문서 유형 결정

`references/문서유형정하기.md`를 따라 가장 적합한 유형을 고른다.

출력에는 아래를 포함한다.

- 추천 문서 유형
- 추천 이유
- 대안 유형이 있다면 1개만 추가
- 해당 유형으로 쓸 때 포함해야 할 핵심 항목

### 3. 정보 구조 개선 또는 초안 구조 생성

`references/정보-구조-만들기.md`를 따라 구조를 점검한다.

다듬기 모드에서는:

- 한 페이지에 목표가 하나인지 확인
- 개요 유무 확인
- 제목 구조와 용어 일관성 점검
- 가치가 앞에 오는지 확인
- 필요한 경우 문서를 분리/재배치

생성 모드에서는:

- 독자 질문 순서대로 목차 생성
- 개요를 먼저 배치
- 단계별 흐름과 예시, 점검 섹션 추가
- 사용자가 바로 실행하거나 학습할 수 있게 `왜 하는지`도 함께 설명

### 4. 문장 다듬기

`references/문장-다듬기.md`를 따라 문장을 개선한다.

반드시 적용할 것:

- 한 문장에 하나의 핵심
- 메타 담화 최소화
- 구체적인 동사 사용
- 용어 표기 일관성 유지
- 자연스러운 한국어 표현 유지
- 수동형보다 능동형 우선
- 코드, 경로, 명령은 구분해서 제시

### 5. 결과 출력

#### 다듬기 모드 출력 형식

- 추천 문서 유형
- 진단 요약
- 구조 개선 포인트
- 문장 개선 포인트
- 다듬은 최종본
- 남은 주의점

#### 생성 모드 출력 형식

- 추천 문서 유형
- 독자와 목표
- 개요
- 추천 목차
- 본문 초안
- 예제 또는 명령어
- 자주 막히는 지점 / 체크리스트

## 예시 1: 기존 문서 다듬기

사용자 요청:

```text
이 문서 다듬어줘. 독자는 Flutter 초보 개발자야.
```

해야 할 일:

- 문서 유형 추천
- 구조 문제 진단
- Step 2 기준으로 목차 재정리
- Step 3 기준으로 문장 다듬기
- 최종 개선본 제시

## 예시 2: 새 문서 생성

사용자 요청:

```text
flutter 사용법 문서 만들어줘
```

해야 할 일:

- 본문이 없으므로 생성 모드로 전환
- 추천 문서 유형은 기본적으로 `시작하기 가이드` 또는 `튜토리얼` 중 선택
- 독자가 빠르게 실행할 수 있는 흐름으로 개요, 설치, 프로젝트 생성, 실행, 다음 단계까지 포함
- 마지막에 체크리스트나 자주 생기는 문제를 덧붙임

## 출력 톤

- 이모지는 사용하지 않는다.
- 문장은 명확하고 단정적으로 쓴다.
- 설명은 친절하되 장황하게 늘어놓지 않는다.
- 여러 개선 옵션을 따로 나열하기보다 하나의 좋은 안으로 통합한다.

## 금지 사항

- 사용자 입력 없이 근거 없는 버전, 수치, API 세부사항을 지어내지 않는다.
- 참조 문서인데 튜토리얼처럼 장황하게 쓰지 않는다.
- 문제 해결 문서인데 배경 설명만 길게 늘어놓지 않는다.
- 문장을 다듬으면서 의미를 임의로 바꾸지 않는다.
